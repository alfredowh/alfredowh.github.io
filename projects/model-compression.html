Model Compression: Pruning & Knowledge Distillation

M.Sc. Artificial Intelligence, TH Ingolstadt · Course Project

  • Pruning
  • Knowledge Distillation
  • MobileNetV2
  • PyTorch

Overview

  • Goal: find the best trade-off between model size, inference speed, and accuracy on MobileNetV2 via channel-level pruning, then recover accuracy with fine-tuning.
  • Model: MobileNetV2, 3.5M parameters — pretrained on ImageNet1k (1,000 classes), 71.87% Top-1 / 90.31% Top-5.
  • Challenge: MobileNetV2 is already compact by design, leaving little redundancy to prune before accuracy degrades sharply.

View repository →

Pruning Methodology

Structured, channel-level pruning of MobileNetV2's conv layers — speeds up inference on standard hardware, no sparse-matrix support needed.

CriterionIdea
Magnitude-based Rank by L1/L2 weight norm.
Scaling-based Rank by batch-norm scale.
TypeIdea
Layer-wise Every layer pruned by the same fixed ratio.
Global One threshold from the ratio applied over all layers' pooled, sorted criterion values.

Pruning alone collapses accuracy

  • All 4 variants destroyed accuracy immediately with no retraining.
  • → Fine-tuning after pruning is essential.

Fine-Tuning

  • Retraining — plain fine-tuning at a fixed pruning ratio.
  • Iterative pruning — alternate pruning and retraining in stages to reach a target ratio gradually.
  • Knowledge distillation — retrain with the unpruned model as teacher.

Setup — keeping the comparison fair

Every experiment shared a fixed training budget and configuration, so differences reflect the method, not training randomness:

  • Same fixed data split — ImageNet1k-val (50,000 images), split 80/20 train/test once and reused everywhere
  • Same total epoch budget — 20 epochs total for every strategy.
  • Same hyperparameters — 1/10th original learning rate, batch size 16.
  • No data augmentation — removes an extra source of run-to-run randomness.

Results on Retraining: Criterion & Type

Scaling-based vs. magnitude-based

  • Global scaling-based pruning is better than magnitude-based.

Global vs. uniform

  • Global pruning outperforms uniform (layer-wise), despite a lower total parameter count.
  • Global pruning preserves more channels in shallow layers.

Comparing Fine-Tuning Strategies

  • Knowledge distillation: strongest boost across most pruning ratios.
  • Iterative pruning: advantage only shows up at large pruning ratios.

Improvement Potential

Overfitting from limited fine-tuning data

  • Adding horizontal-flip augmentation and a larger batch size recovered noticeably more accuracy at the same pruning ratio.
  • Points to overfitting: fine-tuning here uses ImageNet1k-val (50,000 images) vs. 1.28M in the full training set.

Inference speed

  • CPU latency improves fairly consistently with pruning ratio.
  • GPU improvement is somehow inconsistent (probably due to irregular channel counts after pruning not aligning well with the GPU's tiling/warp architecture).

Conclusion

  • MobileNetV2 is highly sensitive to pruning — channel-level pruning is aggressive for an already-compact model.
  • Global Scaling-based pruning combination beats other pruning strategies.
  • Iterative pruning and knowledge distillation improve accuracy over one-shot retraining.
  • Further hyperparameter tuning and more training data could yield additional improvements.
